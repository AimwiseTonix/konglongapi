# 七牛云热更新和机器码授权

## 你需要准备什么

1. 七牛云对象存储 bucket，并把 `tonixx.aimwise.cn` 绑定到这个 bucket。
2. 一个可运行接口的授权服务器域名，例如 `https://license.aimwise.cn`。
3. 每个客户一个授权码，服务端保存授权码、可绑定机器数量、到期时间和已绑定机器码。

七牛云域名只能放静态更新文件，不能安全地做授权状态数据库。机器码绑定必须由服务器判断，否则只放在客户端里很容易被改掉。

## 部署授权服务器到 Vercel

可以把这个项目推到 GitHub，然后在 Vercel 里导入这个仓库。Vercel 负责跑 `api/license/*` 里的授权接口。

Vercel 函数本身不能可靠保存本地文件，所以授权数据需要放到 Redis。推荐用 Upstash Redis，Vercel 集成或 Upstash 官网都可以创建。

### 1. 准备 GitHub 仓库

把项目推到你的 GitHub 仓库。注意不要上传这些本机文件：

- `.env.qiniu.local`
- `license-db.json`
- `release/`
- `node_modules/`

这些已经在 `.gitignore` 里处理了。

### 2. 在 Vercel 导入项目

进入 Vercel，选择 `Add New Project`，导入你的 GitHub 仓库。项目框架可以让 Vercel 自动识别；这个项目的授权接口在 `api/` 目录，Vercel 会自动部署为 Serverless Functions。

### 3. 配置 Redis

创建 Upstash Redis 后，把下面两个环境变量填到 Vercel 项目 Settings -> Environment Variables：

```text
UPSTASH_REDIS_REST_URL=你的 Upstash REST URL
UPSTASH_REDIS_REST_TOKEN=你的 Upstash REST TOKEN
```

### 4. 配置授权密钥

再添加两个环境变量：

```text
LICENSE_TOKEN_SECRET=一串很长的随机字符串，用来签发客户本地授权 token
LICENSE_ADMIN_TOKEN=一串很长的随机字符串，用来管理授权码
```

`LICENSE_ADMIN_TOKEN` 只给你自己本地管理脚本用，不能发给客户。

### 5. 部署后测试

部署完成后，访问：

```text
https://你的项目.vercel.app/api/license/health
```

如果返回 `{"ok":true}`，授权服务器就通了。

### 6. 让桌面软件连接授权服务器

修改 `electron/app-config.js`：

```js
export const LICENSE_SERVER_URL = 'https://你的项目.vercel.app'
```

然后重新打包发版：

```powershell
npm run release:qiniu
```

### 7. 在线生成授权码

部署后，你可以在本机用远程命令生成授权码：

```powershell
$env:LICENSE_SERVER_URL="https://你的项目.vercel.app"
$env:LICENSE_ADMIN_TOKEN="你的 LICENSE_ADMIN_TOKEN"
npm run license:remote:create -- --days=365 --machines=1 --note="客户名或订单号"
```

查看线上授权码：

```powershell
npm run license:remote:list
```

停用授权码：

```powershell
npm run license:remote:disable -- --key=授权码
```

客户换电脑时清空绑定：

```powershell
npm run license:remote:reset-machines -- --key=授权码
```

## AccessKey / SecretKey 在哪里

进入七牛云控制台，点右上角头像或账号入口，找到“密钥管理”或“Key 管理”，里面会显示 AccessKey 和 SecretKey。SecretKey 只用于你自己上传文件，不能写进软件，也不要发给客户。

如果暂时找不到密钥，可以先在七牛云对象存储控制台手动上传更新文件，不影响软件自动更新。

## 七牛云文件夹怎么建

对象存储里没有真正的文件夹，上传文件时把对象名写成带前缀即可，例如：

- `jushou-writer/latest.yml`
- `jushou-writer/jushou-writer-0.1.1-x64.exe`
- `jushou-writer/jushou-writer-0.1.1-x64.exe.blockmap`

软件会从这里检查更新：

`https://tonixx.aimwise.cn/jushou-writer/latest.yml`

## 发新版本流程

1. 修改 `package.json` 里的 `version`，例如从 `0.1.0` 改成 `0.1.1`。
2. 运行：

```powershell
npm run dist:win
```

3. 把 `release` 目录里生成的这些文件上传到七牛云的 `jushou-writer/` 前缀下：

- `latest.yml`
- `.exe` 安装包
- `.blockmap`

4. 已安装的客户下次打开软件时，会自动检查 `latest.yml`，发现新版本后自动下载并提示重启安装。

也可以让脚本自动打包并上传：

```powershell
npm run release:qiniu
```

如果已经打包好，只想重新上传当前 `release` 目录：

```powershell
npm run upload:qiniu
```

上传脚本读取 `.env.qiniu.local`。这个文件只放在本机，不要提交或发给别人。当前 `tonixx.aimwise.cn` 绑定的 bucket 是 `shorttest`，脚本会上传到 `shorttest/jushou-writer/`。

上传前可以先检查会推送哪些文件：

```powershell
npm run upload:qiniu:check
```

## 授权服务器最小接口

客户端会请求两个接口：

- `GET /api/license/health`
- `POST /api/license/activate`
- `POST /api/license/validate`

请求里会包含：

- `licenseKey`：客户输入的授权码
- `machineCode`：本机机器码
- `appVersion`：软件版本

项目里有一个最小示例：`server/license-server.example.js`。它可以帮你理解流程，但正式销售建议部署到云服务器，并把授权码数据库换成真正的数据库。

## 生成授权码

现在可以先用本地命令生成授权码。授权码只授权软件本身，不包含 API Key，不包含模型额度。

生成一个 365 天、只允许绑定 1 台机器的授权码：

```powershell
npm run license:create -- --days=365 --machines=1 --note="客户名或订单号"
```

生成永久授权码：

```powershell
npm run license:create -- --machines=1 --note="客户名或订单号"
```

查看所有授权码：

```powershell
npm run license:list
```

停用授权码：

```powershell
npm run license:disable -- --key=XXXX-XXXX-XXXX-XXXX-XXXX-XXXX
```

恢复授权码：

```powershell
npm run license:enable -- --key=XXXX-XXXX-XXXX-XXXX-XXXX-XXXX
```

客户换电脑时，清空这张授权码已绑定机器：

```powershell
npm run license:reset-machines -- --key=XXXX-XXXX-XXXX-XXXX-XXXX-XXXX
```

本地授权码数据库文件是 `license-db.json`。如果你运行示例授权服务器 `npm run license:demo`，它会读取同一个文件。

桌面端打包时，需要让主进程知道授权服务器地址：

```powershell
$env:LICENSE_SERVER_URL="https://license.aimwise.cn"
npm run dist:win
```

也可以直接修改 `electron/app-config.js`：

```js
export const LICENSE_SERVER_URL = 'https://license.aimwise.cn'
```

如果不配置这个地址，软件会显示机器码，但无法真正联网激活。

开发调试时不会强制授权；打包后的安装包会强制授权。若想在源码模式下也测试拦截效果，可以这样启动：

```powershell
$env:LICENSE_REQUIRED="1"
.\start-desktop.bat
```
